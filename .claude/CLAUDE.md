### Markdown 笔记格式规范

#### 1. 基础结构


#### 2. 代码块规范

- **代码块必须指定语言**，如 `javascript`, `typescript`, `css`, `html`
- **代码应附带解释**，避免纯代码堆砌
- **代码应贴合实际场景**，避免过于抽象的示例

```javascript
// ✅ 好的示例：实际场景中的防抖函数
function debounce(func, delay) {
  let timer = null;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      func.apply(this, args);
    }, delay);
  };
}

// 应用场景：搜索框输入防抖
const searchInput = document.querySelector("#search");
searchInput.addEventListener(
  "input",
  debounce((e) => {
    console.log("搜索:", e.target.value);
  }, 500),
);
```

#### 3. 重点内容标记

- **重要概念**：使用加粗 `**重要概念**`
- `代码标识`：使用反引号包裹
- ~~错误理解~~：使用删除线标记错误
- > 提示：使用引用块标注提示信息

#### 4. 列表使用规范

- 有序列表用于步骤说明
- 无序列表用于要点列举

```markdown
1. 第一步：初始化状态
2. 第二步：执行操作
3. 第三步：更新状态

- 核心要点 1
- 核心要点 2
  - 子要点 2.1
```

#### 5. 表格使用

用于对比和总结：

| 特性 | 优点         | 缺点         | 适用场景   |
| ---- | ------------ | ------------ | ---------- |
| 防抖 | 减少调用频率 | 可能延迟响应 | 输入框搜索 |
| 节流 | 稳定调用频率 | 可能遗漏调用 | 滚动事件   |

#### 6. 注意事项

- **事实核查**：所有技术内容必须经过搜索验证，确保准确性
- **中文标点**：使用中文标点符号（，。：；？！）
- **代码注释**：关键代码必须添加中文注释
- **引用来源**：如有参考外部资料，应在文末标注
- **面试导向**：知识要点以面试常见问题为中心，聚焦高频考点和易错点
- **技术三要素**：对于技术概念，必须涵盖三要素：它是什么、为什么选择这项技术、有什么应用场景
- **减少口语化**：避免使用"简单来说"、"你可以理解为"、"比如"等口语化表述，改用客观陈述；删除"你"、"我"等人称代词，使用第三人称客观描述；避免使用"门卫"、"翻译 + 管家"等生活化比喻，使用专业技术术语
- **代码配合场景**：配合合适的代码段解释应用场景，代码示例应直接对应三要素中的「应用场景」
- **不用 Emoji**：笔记中除非绝对必要，否则不使用 Emoji，保持内容专业简洁
- **搜索确认**：对于知识点或面试经验中常问的内容，必须先搜索确认问题再编写对应内容，确保内容准确且没有歧义
